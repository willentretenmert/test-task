Необходимо написать распределенный сервис, которые определяет банк по БИНу карты.

На вход сервису подается номер карты, сервис должен вернуть название банка, которомупринадлежит данная карта.

Можно использовать сторонние сервисы (ссылка1, ссылка2), базы данных, кэш или любые другие технологии
    которые посчитаете уместными (желательно обосновать).
База БИНов неопознанных карт должна пополняться динамически: если сервис по имеющимся в нем данным не может определить банк,
    можно пойти в сторонний сервис для получения информации по неизвестному ранее БИНу.
    Сервис должен “запомнить” полученную информацию и при последующих обращениях по этому БИНу брать данные из своего источника.
По окончанию необходимо провести нагрузочное тестирование (tank, k6 или любой другой сервис/способ для нагрузочного),
    также необходимо сформировать отчет о проведенном тестировании).

Решений этой задачи существует много, вам необходимо проявить фантазию и решить задачу качественно.
Требования к сервису:
 • Сервис должен выдерживать 1000 rps, с максимальным временем отклика 500ms (95p)
 • Сервис должен содержать в себе конфиги для развертки в docker-compose.

Срок выполнения основной задачи до 3-х рабочих дней

Что делать после:
    Залить на гитхаб/гитлаб сервис. В .md файлах описать принцип работы и прикрепить туда же отчет о нагрузочном тестировании.
Скинуть ссылку на ревью в тг @fprocc

Доп. задачи:
 • Написать балансировщик
 • Несколько инстансов

Срок выполнения доп. задач до 2-х рабочих дней

Проверка работоспособности сервиса:

Что необходимо для проверки:
 • Endpoint который принимает сервис должен быть GET "/bin?bin={bin_here}"
 • Установить k6 и запустить скрипт, изменив порт "{your_service_port}" на ваш, который вы задали
 • Сохранить скрипт в файле  .js
